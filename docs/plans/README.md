# plans/

How we will build something — phased work and migrations.

**Put here:**
- Phased migration plans, sequenced work, rollout strategies.
- Status trackers for in-progress multi-PR efforts.

**Do not put here:**
- The decision and rationale that produced the plan — `design/`.
- One-shot tasks or quick fixes — open a GitHub issue instead.

**Naming convention:** `<yyyy-mm-dd>-<topic>.md`
Examples: `2026-05-15-ports-layer.md`.

**Allowed `status:` values:** `Draft`, `In-Progress`, `Complete`, `Abandoned`.

Plans are archived in place once executed — flip the status, do not delete the file.
